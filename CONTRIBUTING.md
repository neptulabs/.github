# Contributing to Neptu / Neptune Framework

Thanks for your interest in contributing!

## Ways to contribute

- **Bug reports** – If something is broken, open an issue using the *Bug report* template.
- **Feature requests** – If you have an idea for the framework, use the *Feature request* template.
- **Documentation improvements** – Typos, clarifications, and examples are always welcome.

Before opening an issue, please search existing issues to avoid duplicates.

## Development workflow

For code changes:

1. Fork the repository and create a feature branch.
2. Run tests locally (see the README for commands).
3. Open a pull request with:
   - A clear description of the change
   - Any related issue numbers
4. A maintainer will review your PR. Please be willing to address review comments.

## Coding guidelines

- Follow the existing project style and patterns.
- Keep changes as small and focused as possible.
- Write or update tests when you change behavior.
- Do not include secrets, credentials, or PHI in code, tests, or examples.

## Security

Please **do not** report security issues in public GitHub issues. See `SECURITY.md` for how to report vulnerabilities.
